# AI 면접관 (CS 기술 면접 준비 서비스) 기획서

> 최종 업데이트: 2026-07-01
> 상태: 기획 확정 (구현 전)

## 1. 목적

IT 기업 취업을 준비하며 CS 기술 면접을 연습할 수 있는 **AI 면접관 웹 서비스**를 만든다.
단순히 예상 질문 리스트를 보여주는 도구가 아니라, **실제 면접처럼 사용자의 답변을
읽고 그 안을 파고드는 꼬리질문**을 이어가는 것이 핵심이다.

## 2. 목표 (이 프로젝트가 노리는 두 가지)

1. **실전 연습 도구** — 본인이 실제 면접처럼 CS 질문에 답하고, 꼬리질문에 대응하고,
   끝나면 피드백을 받는다.
2. **취업 포트폴리오** — 백엔드(Spring Boot) 역량, LLM 제어(프롬프트 엔지니어링),
   인증/DB 설계 등을 보여주는 프로젝트로 활용한다.

> 두 목표는 상충하지 않는다. "실전에 쓸 만큼 잘 동작하는 완성품"이 곧 좋은 포트폴리오다.

## 3. 대상 사용자

- CS 기술 면접을 준비하는 취업준비생 (1차 사용자 = 개발자 본인)
- 넓게는 신입/주니어 개발자 지망생

## 4. 핵심 컨셉 — "질문 DB + 유연한 LLM 꼬리질문" (패턴 B)

- **첫 질문(오프닝)**: 카테고리별 질문 DB에서 꺼낸다. 난이도·주제를 통제할 수 있다.
- **꼬리질문**: 사용자 답변을 LLM에 넘겨, 해당 카테고리의 질문 풀을 **참고자료로 함께
  주입**한 뒤 생성한다. DB에 갇히지 않고 답변 맥락에 맞춰 유연하게 변형·생성한다.
- **범위 이탈 완충**: "참고 질문 풀에서 크게 벗어나지 말되, 답변에 명백한 오류·모순이
  있으면 그것을 우선 파고든다"는 규칙으로 산으로 가는 것을 막는다.
- **평가 리포트**: 세션 종료 후 개념별 점수 + 놓친 키워드 + 모범답안을 제공한다.

즉 DB의 역할은 "첫 질문 저장소"가 아니라 **"해당 카테고리의 지식/질문 풀"**이다.

## 5. 기능 명세

### 5.1 MVP (1차) — 반드시 동작해야 하는 것

- **세션 설정**: 카테고리 선택(단일/다중 선택, 전체 랜덤), 질문 수, 난이도
- **질문 진행**: DB에서 첫 질문 제시 → 사용자 텍스트 답변 → LLM 꼬리질문 1~2개
- **답변 입력**: 텍스트 타이핑
- **평가 리포트**: 개념별 정확성/깊이 점수(5점 척도), 놓친 핵심 키워드, 모범답안
- **인증/기록**: 회원가입·로그인(Spring Security + JWT), 사용자별 면접 세션 기록 저장

### 5.2 2차 (확장) — MVP 안정화 후

- **음성 입력**: STT(음성→텍스트) 연동으로 실제 면접처럼 말하기
- **카테고리 확장 (B안)**: Java, Web, Spring 등 언어·프레임워크 질문 추가
- **소셜 로그인**: OAuth2(Google/Kakao 등)
- **기록 활용**: 지난 세션 다시보기, 취약 개념 통계·추천

## 6. 다룰 범위 (카테고리)

### MVP = A안 (핵심 CS)
- 핵심 6개: **자료구조 / 알고리즘 / 운영체제 / 네트워크 / 데이터베이스 / 디자인패턴**
- 확장 3개: **컴퓨터구조 / 소프트웨어공학(OOP·TDD 등) / 개발상식(REST·Git 등)**

### 2차 = B안 (직무 밀착)
- **Java / Web / Spring** 등. (지원 직무가 백엔드라 포트폴리오 강점이 됨)

> 카테고리는 코드에 하드코딩하지 않고 데이터(테이블)로 관리하여, B안 확장 시
> 스키마 변경 없이 데이터만 추가하면 되도록 설계한다.

## 8. 결과물 형태 & 기술 방향 (요약)

- **형태**: 웹 앱
- **프론트엔드**: React
- **백엔드**: Spring Boot
- **DB**: H2(개발 초기) → 로컬 Docker PostgreSQL(주력) → AWS RDS(배포)
- **LLM**: 외부 LLM API 호출 (꼬리질문 생성 + 평가)

자세한 기술 구조는 `아키텍처.md`, 면접관 프롬프트 규칙은 `프롬프트-설계.md`,
결정 근거는 `결정사항.md` 참고.

## 9. 개발 진행 방식

1. **기획·설계 확정** (현재 단계) — 이 문서들 작성
2. **질문 데이터 수집** — 소스별·카테고리별로 질문 추출 → JSON/CSV
3. **구현** — Claude Code로 이동하여 Spring Boot + React 개발, `docs/`에 문서 누적 갱신
4. **DB 적재 → 배포**
